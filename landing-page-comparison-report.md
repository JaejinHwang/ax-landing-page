# Palantir vs 뤼튼 AX 랜딩 페이지 비교 분석

> 분석 일자: 2026-01-27
> 목적: 새로운 AX 랜딩 페이지 기획 시 참고할 디자인/구조적 인사이트 도출

---

## 1. 유사점

두 사이트 모두 **B2B AI 솔루션** 기업의 랜딩 페이지로, 공통적인 설계 철학이 관찰됩니다.

| 공통 요소 | 설명 |
|-----------|------|
| **다크 모드 기반** | 두 사이트 모두 어두운 배경을 주조색으로 사용하여 기술 기업 이미지를 형성 |
| **단일 강조색** | Palantir는 흰색 대비, 뤼튼 AX는 티얼(`#09BBA1`) — 모두 한 가지 포인트 컬러만 사용 |
| **대형 타이포 헤딩** | Hero에서 40px 이상의 대형 텍스트로 핵심 메시지 전달 |
| **스크롤 등장 애니메이션** | `fadeInUp` 패턴으로 스크롤 시 콘텐츠가 순차적으로 나타남 |
| **파트너/고객 사회적 증거** | 고객사 후기 또는 파트너 추천사를 섹션으로 구성 |
| **Sticky Header** | 상단 고정 네비게이션 + 우측 CTA 버튼 |
| **미니멀 UI** | 장식적 요소를 최소화하고 콘텐츠 중심으로 구성 |

---

## 2. 핵심 차이점 및 장단점 비교

### 2.1 페이지 구조

| 항목 | Palantir | 뤼튼 AX |
|------|----------|---------|
| **페이지 방식** | 멀티 페이지 (메인 + 하위 페이지 링크) | **싱글 페이지** (앵커 네비게이션) |
| **총 높이** | ~6,800px (약 7뷰포트) | ~13,550px (약 14뷰포트) |
| **네비게이션** | 햄버거 메뉴 → 풀스크린 오버레이 | 상단 인라인 메뉴 (6개 항목 노출) |
| **정보량** | 핵심만 요약, 상세는 하위 페이지 | 한 페이지에 모든 정보 수록 |

**판단**: 뤼튼 AX의 **싱글 페이지 + 인라인 메뉴** 방식이 B2B 리드 생성(문의 전환) 목적에 더 적합합니다. 방문자가 이탈 없이 한 흐름으로 정보를 소비하고 문의까지 도달할 수 있습니다. 다만 뤼튼 AX는 14뷰포트 분량으로 다소 길어서, 핵심 정보 위주로 압축할 여지가 있습니다. Palantir 수준(7뷰포트)이 스크롤 피로도 측면에서 더 낫습니다.

### 2.2 Hero 섹션

| 항목 | Palantir | 뤼튼 AX |
|------|----------|---------|
| **배경** | 풀스크린 루프 비디오 | 정적 그래픽 + CSS 글로우 애니메이션 |
| **메시지** | "AI-Powered Automation for Every Decision" (범용적) | "대한민국 1등 AI 뤼튼, 당신의 AX를 책임집니다" (구체적) |
| **CTA** | 없음 (Scroll to Explore) | 2개 버튼 (문의하기, AX Report) |

**판단**: 뤼튼 AX의 접근이 더 효과적입니다. Hero에서 바로 CTA를 제공하고, 메시지가 "누가(뤼튼) 무엇을(AX) 해주겠다"라는 구체적 가치 제안을 담고 있습니다. 다만 Palantir의 비디오 배경은 시각적 몰입감에서 우위이므로, 짧은 데모 영상이나 모션 그래픽을 검토할 가치가 있습니다.

### 2.3 신뢰 구축 방식

| 항목 | Palantir | 뤼튼 AX |
|------|----------|---------|
| **후기 형태** | 25개 이상 고객 직접 인용 (자동 스크롤 marquee) | 3대 클라우드 파트너(MS, AWS, GCP) 추천사 캐러셀 |
| **로고 노출** | 없음 (후기 텍스트에 회사명만 표기) | 24개 클라이언트 로고 그리드 |
| **실적 수치** | 없음 (제품 중심 서술) | 1300억+ 투자, 500만 MAU, CES 수상 등 |

**판단**: 뤼튼 AX가 **신뢰 구축 장치를 더 다층적으로** 설계했습니다. 글로벌 빅테크 추천사 + 국내 유명 로고 + 정량적 실적의 3중 구조가 효과적입니다. Palantir의 대량 고객 인용은 스케일감을 보여주지만, 읽히지 않고 흘러가는 marquee 형태라 실질적 설득력은 떨어집니다. **핵심 파트너 3~5개의 심층 추천사 + 로고 그리드 + 핵심 수치**의 조합이 가장 효과적입니다.

### 2.4 서비스/제품 설명

| 항목 | Palantir | 뤼튼 AX |
|------|----------|---------|
| **제품 소개** | 5개 플랫폼을 한 줄 설명 + 넘버링 (/0.1~0.5) | 3개 서비스(교육/컨설팅/개발)를 각각 독립 섹션으로 |
| **깊이** | 얕음 (클릭 → 하위 페이지) | 깊음 (각 섹션에 카드, 탭, 아코디언으로 상세 내용 포함) |
| **차별화 기술** | 없음 (메인 페이지에서는 미노출) | AutoBE, AutoView 기술을 아코디언으로 상세 설명 |

**판단**: 뤼튼 AX가 **한 페이지 안에서 의사결정에 필요한 정보를 충분히 제공**합니다. 특히 "무엇을 해주는가"(서비스)와 "어떻게 하는가"(기술)를 분리하여 배치한 구조가 명확합니다. Palantir의 간결한 리스트는 미학적으로 세련되지만, 방문자가 구매 결정을 내리기에는 정보가 부족합니다.

### 2.5 디자인 퀄리티

| 항목 | Palantir | 뤼튼 AX |
|------|----------|---------|
| **폰트** | 커스텀 폰트 Alliance No.1/2 (독자적 정체성) | Pretendard (한국어 최적화, 범용적) |
| **시각 장치** | 풀스크린 비디오, 대형 이미지, 무한스크롤 캐러셀 | CSS 글로우, 글래스모피즘 카드, 데모 스크린샷 |
| **레이아웃** | 다크↔라이트 교차 리듬 | 전체 다크 모드 일관 |
| **넘버링/인덱싱** | `/0.1`~`/0.5` 인덱스 표기 | 없음 |
| **디자인 토큰** | CSS 변수 체계적 관리 | CSS 변수 (`--cp1` 등) 부분 사용 |

**판단**: 시각적 세련도는 Palantir이 한 단계 위입니다. 특히 **다크↔라이트 교차 리듬**이 장시간 스크롤 시 시각적 피로도를 줄여줍니다. 뤼튼 AX의 전체 다크 모드는 통일감은 있지만 단조로울 수 있습니다. 글래스모피즘 카드는 트렌디하지만 시인성 측면에서 콘텐츠 구분이 약해지는 단점이 있습니다.

### 2.6 전환(Conversion) 설계

| 항목 | Palantir | 뤼튼 AX |
|------|----------|---------|
| **CTA 수** | 2개 (Request a Demo, Start Building) — 페이지 최하단 | 5개 (문의하기 헤더, Hero, 교육, 컨설팅, 개발) — 곳곳에 배치 |
| **CTA 위치** | 페이지 끝 | Hero + 각 섹션 하단 반복 배치 |
| **연락 채널** | 별도 페이지 링크 | 같은 페이지 내 `#solution-form` 앵커 |

**판단**: 뤼튼 AX의 전환 설계가 **B2B 리드 생성에 압도적으로 유리**합니다. 각 서비스 섹션마다 "교육 문의하기", "컨설팅 문의하기" 등 맥락에 맞는 CTA를 배치하여, 방문자가 관심이 생긴 시점에 바로 전환할 수 있습니다. Palantir처럼 CTA를 페이지 끝에만 두면 대부분의 방문자가 도달하지 못합니다.

## 3. 새로운 AX 랜딩 페이지 기획 시 고려사항

### 3.1 구조 설계

- **싱글 페이지 + 7~10뷰포트 분량**이 이상적 (뤼튼 AX의 14뷰포트는 과도, Palantir의 7뷰포트가 적정)
- **앵커 기반 인라인 네비게이션** 유지 — 방문자가 원하는 정보에 바로 접근 가능
- **섹션별 CTA 반복 배치** — Hero, 각 서비스 섹션 하단, 페이지 최하단 최소 3~4회
- **정보 흐름**: `문제 제기 → 해결 방법 → 증거(신뢰) → 전환`의 설득 구조 준수

### 3.2 시각 설계

- **다크 기반 + 부분 라이트 섹션 교차** 적용 — 전체 다크의 통일감과 교차 배치의 리듬감을 모두 확보
- **브랜드 강조색을 글로우/그라디언트로 활용** — 뤼튼 AX의 티얼 글로우 효과가 기술 브랜드에 적합
- **Hero에 모션 요소 추가** — 정적 그래픽보다 짧은 루프 모션(Lottie 또는 CSS 애니메이션)이 몰입감 향상
- **글래스모피즘은 제한적으로** — 핵심 카드에만 적용하고, 나머지는 명확한 배경색 차이로 구분

### 3.3 콘텐츠 설계

- **Hero 메시지는 구체적 가치 제안** — "당신의 AX를 책임집니다" 수준의 직접적 표현이 효과적
- **신뢰 장치 3중 구조** 유지: 파트너 추천사(빅테크) + 클라이언트 로고 + 핵심 수치
- **서비스별 독립 섹션** — 각 서비스의 대상, 내용, CTA를 한 곳에 자기완결적으로 구성
- **기술 차별점은 인터랙티브하게** — 아코디언/탭 패턴으로 관심 있는 사람만 깊이 들어갈 수 있도록
- **FAQ는 필수** — B2B 의사결정자의 공통 우려(보안, 비용, 온프레미스 등)를 선제적으로 해소

### 3.4 인터랙션 설계

- **스크롤 등장**은 `fadeInUp` + stagger 딜레이 (0.6~0.8s ease-out)가 검증된 패턴
- **CTA 호버**는 `translateY(-2px)` + 브랜드색 글로우 `box-shadow`로 클릭 유도
- **아코디언** 트랜지션은 `height + opacity + padding` 3중 전환 (0.3~0.35s)
- **캐러셀보다는 정적 그리드**를 우선 검토 — 자동 스크롤 캐러셀은 사용자가 제어감을 잃어 오히려 정보 전달 효율이 떨어질 수 있음
